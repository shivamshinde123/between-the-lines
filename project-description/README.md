# Between the Lines Project Description

This folder stores persistent project context for the "Between the Lines" web app so future sessions can recover requirements, workflow, and decisions quickly.

## Files

- `product-overview.md`: concise product summary and goals
- `requirements.md`: feature requirements and constraints
- `architecture.md`: current technical direction and implementation assumptions
- `roadmap.md`: phased implementation breakdown
- `workflow.md`: required GitHub and delivery workflow
- `open-questions.md`: unresolved decisions that block implementation details
- `progress-log.md`: session-by-session progress summary
